Função responsável por enviar o kernel para execução, ler o resultado da operação, e transferir os dados de volta ao hospedeiro.

[in]globalWorkSize- tamanho para cada dimensão do espaço de índices.
[in]localWorkSize- Arrau de tamanhos dos grupos de trabalho.
[in]buffer- objeto de memória do tipo buffer a ser lido.
[in]blocking_read- Define se a leitura é blockeada ou não.
Operador Descrição
cl_TRUE Booleano correspondete a verdadeiro.
cl_FALSE Booleano correspondete a falso.
[in]datasize- comprimentos em bytes dos dados a serem transferidos
[in]ptr- região do host onde os dados serão escritos

◆ lopcl_CreateBuffer()

cl_mem lopcl_CreateBuffer ( size_t  lopcl_DATASIZE,
cl_mem_flags  lopcl_FLAGS,
cl_bool  lopcl_FLAG1,
void *  host_ptr 
)

Função responsável por criar os buffers de memória, empilhar, e tornar disponível ao dispositivo os dados do hospedeiro.

Parâmetros
[in]lopcl_DATASIZE- Tamanho em bytes do objeto de memória que será usado.
[in]lopcl_FLAGS- Campo de bits que é usado pra especificar informações sobre alocação e uso, como a área de memória deve ser usada para alocar o elemento.
Identificador Descrição
CL_MEM_READ_WRITE Essa flag especifica que a memória será de leitura e escrita para o kernel. Essa é o padrão.
CL_MEM_WRITE_ONLY Essa flag especifica que a memória será somente de escrita para o kernel. CL_MEM_READ_WRITE e CL_MEM_WRITE_ONLY são mutuamente exclusivas.
CL_MEM_READ_ONLY Essa flag especica que a memória será somente de leitura para o kernel. CL_MEM_READ_WRITE e CL_MEM_READ_ONLY são mutuamente exclusivas.
CL_MEM_USE_HOST_PTR Essa flag só é valida se o host_ptr não for NULL. Se usada, a flag especifica que a memória referencida será usada como bits de armazenamento para um objeto de memória.
CL_MEM_ALLOC_HOST_PTR Essa flag especifica que a aplicação deseja que o OpenCL aloque a memória de modo acessível ao host. CL_MEM_USE_HOST_PTR e CL_MEM_ALLOC_HOST_PTR são mutuamente exclusivas.
CL_MEM_COPY_HOST_PTR Essa flag só é valida se o host_ptr não for NULL. Se usada, a flag especifica que a aplicação deseja que a memória seja alocada e copiada para o objeto de memória.
CL_MEM_HOST_WRITE_ONLY Essa flag especifica que somente o host pode escrever no objeto de memória.
CL_MEM_HOST_READ_ONLY > Essa flag especifica que somente o host pode ler o objeto de memória.
CL_MEM_HOST_NO_ACCESS > Essa flag especifica que o host não poderá ler ou escrever no objeto de memória.
lopcl_FLAG1- Indica se a operação de escrita está bloqueada ou não
[in]*host_ptr- Um ponteiro para um buffer de dados que já foi alocado pela aplicação.
Retorna
cl_mem - Que é o onjeto de memória que será usado pela aplicação
